# Google 同步冲突

Chrome 的同步引擎是「书签重分类」场景最大的坑。直接改本地 `Bookmarks` 文件，很可能在 Chrome 启动后被同步撤销。本文记录三种典型表现、触发条件、和正确的应对顺序。

## 三种典型表现

### 表现 1：旧文件夹"复活"
你删除了旧的「百度」「个人收藏」等文件夹、建了新分类。Chrome 启动后，旧文件夹又出现了，和新分类并存。书签栏顶层从你期望的 7 项变成 25 项。

**原因**：同步引擎拿云端记录的旧结构和本地新结构做合并，旧文件夹被当作"云端新增"合并回来。

### 表现 2：书签内容被"抽空"，只剩空文件夹壳
新分类文件夹的名字在，但里面 0 个书签；所有书签又回到了旧文件夹里。

**原因**：你在 Chrome 里手动删过书签（删除事件同步到了云端）。Chrome 启动时认为"用户已删除这些书签"，于是把新分类里的同名书签也清空；同时云端还记录着书签的"原始位置"，把它们放回旧文件夹。

### 表现 3：移动操作被撤销
你把书签从「其他书签」移到「书签栏/某分类」。Chrome 启动后，书签又回到了「其他书签」。

**原因**：同步引擎按节点的「原始位置 GUID」记录。脚本/外部修改移动了节点，但云端的"原位置记录"没变，Chrome 启动时按云端记录"纠正"回来。

## 触发条件

三个条件同时满足才会触发：
1. 用户登录了 Google 账号
2. 书签同步开启（`Preferences` 里 `sync.data_type_status_for_sync_to_signin.bookmarks == true`）
3. 外部脚本修改了 `Bookmarks` 文件（而非通过 Chrome UI）

## 为什么「Chrome 自己执行的操作能同步成功」

Chrome 内部的删除/移动/添加，会生成带时间戳的同步事件，push 到云端，云端接受。
外部脚本改文件，没有同步事件——Chrome 启动时只看到"本地文件和云端不一致"，于是做合并，
而合并倾向于保留云端记录。这就是为什么：

- ✅ **让用户在书签管理器里手动拖动/删除** → 能正确同步
- ❌ **脚本改文件** → 容易被合并撤销

## 正确的应对顺序

### 场景 A：从零开始重分类（推荐流程）

1. **先让用户在 Chrome 内关闭书签同步**（`chrome://settings/syncSetup` → 关闭书签类型）
   - 这一步会断开书签的同步通道，但不删本地书签
2. 用户**完全退出 Chrome**（`⌘+Q`）
3. 脚本备份 + 重写 `Bookmarks` 文件 + 重算 checksum
4. 打开 Chrome，肉眼确认书签栏正确（此时同步已关，不会被撤销）
5. 用户**重新开启书签同步**——此时本地是权威版，Chrome 会把它当作"最新"上传覆盖云端

### 场景 B：重排后发现书签又被清空/合并了

说明同步没关就写了文件。补救：

1. 别慌——书签大概率没丢。检查 `Bookmarks.bak`（Chrome 自动保存的上一版，往往是你写入的完整版）
2. 让用户在 Chrome 里**关闭书签同步** + 退出 Chrome
3. 从 `Bookmarks.bak` 恢复到 `Bookmarks`（重算 checksum）
4. 开 Chrome 验证

### 场景 C：只是少量书签需要换位置

不要用脚本。让用户在书签管理器（`⌘+Shift+O`）里**手动拖动**：
- 选中书签 → 拖到目标文件夹
- 或 `⌘+X` 剪切 → 进目标文件夹 → `⌘+V` 粘贴

Chrome 执行的移动会正确同步，不会被撤销。

## 反复冲突时

如果关了同步、改了文件、开 Chrome、又被合并——说明云端同步状态机和本地持续不一致。终极手段：

1. Chrome 内**关闭同步**并退出
2. 脚本恢复正确的本地文件
3. **保持同步关闭一段时间**（让云端同步状态"冷却"），或直接去 [Google 账户数据面板](https://chrome.google.com/sync) 清空云端同步数据
4. 再重新开启同步

## 检测同步开关的代码

读 `Preferences` 文件：

```python
import json, os
p = os.path.expanduser("~/Library/Application Support/Google/Chrome/Default/Preferences")
d = json.load(open(p))
bm_sync = d.get("sync", {}).get("data_type_status_for_sync_to_signin", {}).get("bookmarks")
# True = 书签同步开启（改文件前必须先关）；False/None = 已关，可安全改文件
```

⚠️ 注意：Chrome 运行时，`Preferences` 在内存里，磁盘版本可能滞后。要改这个开关，**必须在 Chrome 退出后改**，否则会被覆盖。
